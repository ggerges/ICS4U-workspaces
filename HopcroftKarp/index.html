The Hopcroft Karp Algorithm

Key Terms

  • Bipartite Graph

    • A graph that contains vertices with two sets (sets A and B) with edges that connect them and no edges between vertices of the same set

  • A Matching

    • A set of edges without a common vertex. No vertex has the same edge destination in the second set

  • Not a Matching

    • A set of edges sharing common vertices from set a and b. More than one vertex in set a shares the same edge destination in set B

  • Maximul Matching

    • Where there is the maximum amount of edges in the matching with no augmenting paths

  • Augemnting path

    • a path (edges that you can take in a sequence to move around a graph) that starts at an unmatched vertex, and ends at an unmatched vertex

  • Vertex Disjoint

    • Given two paths, the same vertex cannot be present in both paths

  • A Free Vertex

    • A vertex with no matching edges connected to it

  • An Unmatched Vertex

    • A Vertex that is not in the current matching

The Algorithm

Objective

Given a bipartite graph with sets A and B, return the graph with maximal matching.

Steps

  1. Set Matching to empty
  2. Enter Loop and Use Breadth First Search to build an alternating graph, starting at the unmatched vertices of set A. Matching is still empty and all vertices in both sets start unmatched.
  3. Identify unmatched vertices in set B and add path to vertices in set A to current matching until there are no more unmatched vertices. Note that after creating a matching, all paths connected to the unmatched vertex are removed so that there are no disjointed vertices.
  4. Add the matched vertices to our graph and you will notice that not all vertices have a matching, but can be connected to by adding another matching. We can add an augmenting path to connect these vertices.
  5. To do this, we first identify the unmatched Vertices in set B and alternate between matched and unmatched edges until an unmatched vertice in set A is reached by using Depth First Search. If there are no unmatched vertices in either set, you have found the maximal matching and dont need to continue the search.
  6. Remove the matching found in the augmenting path and add the unmatched edges that connected the unmatched vertices in the path.
  7. Loop until there are no more unmatched vertices on both sides and return matching
